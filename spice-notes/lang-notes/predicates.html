Palantir Memo: On Predicates

Q: How can you determine the extent of a predicate?

A: That's why we need the predicate "application" braces :- append( ?x, ?y, ?z ) :- ...

Q: Can we use Prolog-style upper case variables?

A: No. We sometimes need both "?x" and "x" in the same lexical scope. Using "X" and "x" would be poor design.

Q: Why don't we need "?x" variables when writing pattern-matching code such as 

	let Pair( x, y ) := E in B

A: Insisting on a "?" prefix would be more general. However, this is a 1-way unification and the extra generality makes it error prone. Hence we have a simplified and more convenient approach for argument passing and let bindings.

n.b. 1-way unification uses ":=" or "<-". 2-way uses "=".
